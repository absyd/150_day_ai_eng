NumPy arrays & the no-loop philosophy

NumPy stores data in contiguous C arrays. Operations happen in compiled Fortran/C — not Python loops. The golden rule: if you're looping, you're doing it wrong.

Array creation — real workflow patterns
import numpy as np # From data arr = np.array([1, 2, 3], dtype=np.float32) # Structured initialization np.zeros((3, 4)) # all zeros, shape 3×4 np.ones((2, 5)) # all ones np.eye(4) # identity matrix np.arange(0, 10, 0.5) # like range() but floats np.linspace(0, 1, 100) # 100 evenly-spaced pts # Random (set seed for reproducibility) rng = np.random.default_rng(seed=42) rng.normal(0, 1, (1000, 768)) # e.g. embedding matrix
Shape, strides & views — the memory model
Arrays have shape, dtype, and strides. Slices are views (no copy) — critical for performance and a common bug source.
a = np.arange(24).reshape(4, 6) a.shape # (4, 6) a.strides # (48, 8) bytes — row-major C order a.dtype # int64 a.ndim # 2 a.size # 24 total elements # View vs copy — interview classic view = a[1:3, :] # NO copy — modifying view changes a copy = a[1:3, :].copy() # explicit copy # Transpose is free — just flips strides a.T.strides # (8, 48)
Vectorised ops — the no-loop way
# Element-wise arithmetic (no loop needed) scores = np.array([85, 92, 78, 96]) normalised = (scores - scores.mean()) / scores.std() # Boolean masking (replaces if-loops) high_scores = scores[scores > 90] # [92, 96] # Universal functions (ufuncs) — compiled C, extremely fast np.sqrt(scores) np.exp(normalised) np.log1p(scores) # numerically stable log(1+x) # Axis-wise reductions matrix = rng.random((100, 50)) matrix.sum(axis=0) # sum each column → shape (50,) matrix.mean(axis=1) # mean each row → shape (100,) matrix.argmax(axis=1) # index of max per row
Interview insight: "Why is NumPy faster than Python loops?" — Answer: numpy stores homogeneous data in contiguous memory and calls compiled C/Fortran routines. Python loops have per-iteration overhead: type checking, reference counting, bytecode interpretation. NumPy eliminates all of that.

Broadcasting — the shape-matching superpower

Broadcasting lets arrays of different shapes operate together without copying data. NumPy expands dimensions implicitly by the rules below.

The 3 broadcasting rules
1
Pad dimensions on the left
If shapes have different number of dims, prefix 1s. Shape (3,) becomes (1, 3).
2
Expand size-1 dimensions
Any axis of size 1 is stretched to match the other array's size along that axis.
3
Sizes must match or be 1
If neither axis is 1 and sizes differ, you get a ValueError. No silent surprises.

Real-world broadcasting patterns
# Pattern 1: Normalise each row independently X = rng.random((1000, 20)) # 1000 samples, 20 features mean = X.mean(axis=0) # shape (20,) std = X.std(axis=0) # shape (20,) X_norm = (X - mean) / std # broadcast: (1000,20)-(20,) → works! # Pattern 2: Pairwise distance (no loop) a = rng.random((100, 3)) # 100 points in 3D b = rng.random((200, 3)) # 200 points in 3D # a[:,None,:] → (100,1,3) b[None,:,:] → (1,200,3) diffs = a[:, None, :] - b[None, :, :] # (100,200,3) dists = np.sqrt((diffs**2).sum(axis=-1)) # (100,200) # Pattern 3: Bias addition in neural nets W = rng.random((64, 128)) # weight matrix b = rng.random(128) # bias vector (128,) out = X @ W + b # b broadcasts across batch
Interview gotcha: Shape (3,) and (3,1) behave very differently in broadcasting. Always use reshape(-1,1) or [:, np.newaxis] when you need explicit column/row vectors.

Norms — L1, L2, Frobenius

SVD — singular value decomposition
SVD factorises M into U Σ Vᵀ. Real-world uses: dimensionality reduction (PCA), recommender systems, image compression, solving least-squares.
M = rng.random((50, 20)) U, s, Vt = np.linalg.svd(M, full_matrices=False) # U: (50,20) — left singular vectors (data space) # s: (20,) — singular values (importance) # Vt:(20,20) — right singular vectors (feature space) # Rank-k approximation (key idea behind PCA) k = 5 M_approx = U[:, :k] * s[:k] @ Vt[:k, :] # Captures most variance with k components # Explained variance var_explained = s**2 / (s**2).sum() cumulative = np.cumsum(var_explained) # How many components explain 95% variance? n_components = np.searchsorted(cumulative, 0.95) + 1

Zero-loop implementations from scratch

These are the exact implementations interviewers ask for. No for-loops allowed.

Matrix multiply — from scratch
# Naive (loop) — baseline for benchmarking def matmul_naive(A, B): m, k = A.shape k2, n = B.shape assert k == k2 C = np.zeros((m, n)) for i in range(m): for j in range(n): for l in range(k): C[i, j] += A[i, l] * B[l, j] return C # Vectorised (no loops) — the right way def matmul_vec(A, B): return A @ B # or: np.einsum('ik,kj->ij', A, B) # Speed: 500×500 matrices # Naive: ~15 seconds | Vectorised: ~0.003 seconds # Speedup: ~5000×
Cosine similarity — zero loops
# Single pair def cosine_sim(a, b): return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b)) # Batched — all pairs between matrix A and B (no loop!) def cosine_sim_matrix(A, B): # A: (m, d), B: (n, d) → returns (m, n) A_norm = A / np.linalg.norm(A, axis=1, keepdims=True) B_norm = B / np.linalg.norm(B, axis=1, keepdims=True) return A_norm @ B_norm.T # each entry is cosine similarity # Real-world use: semantic search query = rng.random(768) # one query embedding docs = rng.random((10000, 768)) # corpus embeddings sims = cosine_sim_matrix(query[None], docs) # (1, 10000) top_k = np.argsort(sims[0])[::-1][:10] # top-10 indices
Softmax — numerically stable, zero loops
# Naive — WRONG (overflows for large logits) def softmax_naive(x): return np.exp(x) / np.exp(x).sum() # Stable — subtract max first (same result, no overflow) def softmax(x, axis=-1): x = x - x.max(axis=axis, keepdims=True) # subtract max e = np.exp(x) return e / e.sum(axis=axis, keepdims=True) # Batched softmax over rows logits = rng.random((32, 10)) # 32 samples, 10 classes probs = softmax(logits, axis=1) # (32, 10), rows sum to 1 assert np.allclose(probs.sum(axis=1), 1.0) # sanity check

Interview Q&A — NumPy & Pandas

Covering beginner through senior-level questions. Click to expand answers.

What's the difference between a view and a copy in NumPy? ▼
A view shares memory with the original array — modifying the view changes the original. Slicing creates a view. A copy is an independent allocation. Use .copy() to be explicit. The .base attribute tells you if an array is a view. Fancy indexing (integer/boolean arrays) always returns copies. This is a classic interview trap — candidates often assume slices are copies.
Why is np.dot behaving unexpectedly for 3D arrays? ▼
For N-D arrays, np.dot(A, B) is a sum product over the last axis of A and the second-to-last axis of B. This is unintuitive and often wrong. Use @ (matmul) instead — it treats the last two dimensions as matrices and broadcasts over the rest, which is what you almost always want for batched operations.
How would you implement softmax numerically stable? ▼
Subtract the max before exponentiating: x -= x.max(axis=-1, keepdims=True). This shifts all values to be ≤0, so exp() stays in [0,1]. The result is mathematically identical because the max cancels in numerator and denominator. Without this, logits of 1000 cause exp(1000) = inf and you get inf/inf = nan.
When would you use apply() vs vectorised string methods in Pandas? ▼
apply() runs a Python function row-by-row — it's slow because it loops. Always prefer df['col'].str.method() for string ops (vectorised internally), np.where() for conditional logic, and pd.cut()/pd.qcut() for binning. Use apply() only when no vectorised equivalent exists. In interviews, naming np.where as the alternative to apply for conditionals signals senior-level knowledge.
What is the SettingWithCopyWarning and how do you fix it? ▼
It occurs when you chain two indexing operations and try to set a value — Pandas can't guarantee whether you're modifying the original or a copy. Fix: always use df.loc[mask, col] = value for assignments. Or use .copy() to make the intent explicit when creating a subset. This is one of the most common pandas bugs in production code.
What's the difference between transform and apply in groupby? ▼
apply() reduces each group to a scalar or smaller DataFrame — the result has one row per group. transform() returns a result with the same shape as the input — it broadcasts the group result back to every row. Use transform when you want to add a group-level statistic back to each row (e.g., "salary minus department mean"). This pattern is very common in feature engineering.
Explain SVD and one real-world application ▼
SVD decomposes matrix M into U Σ Vᵀ where U and Vᵀ are orthogonal and Σ is diagonal with singular values in descending order. Taking the top-k singular values gives the best rank-k approximation of M (Eckart-Young theorem). Real-world uses: (1) PCA — project data onto principal components for dimensionality reduction; (2) recommender systems — factorize user-item matrix to find latent features; (3) image compression — store k singular vectors instead of full image. Interviewers love this because it connects linear algebra to ML.
How does broadcasting work for pairwise distance computation? ▼
The trick is inserting a new axis to create compatible shapes for broadcasting. For arrays A of shape (m,d) and B of shape (n,d): A[:,None,:] becomes (m,1,d), B[None,:,:] becomes (1,n,d). Subtracting broadcasts to (m,n,d) — every pair of rows. Then sum over the last axis and take sqrt to get the (m,n) distance matrix. This is O(mnd) with no loops, and numpy parallelises it internally. This is the key pattern for KNN, cosine similarity matrices, and attention score computation.
